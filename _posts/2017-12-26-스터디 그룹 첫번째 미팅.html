Working Directory

Working directory 는 일종의 “작업장”과 같은 개념입니다. R과 같은 오픈소스는 많은 경우에 직접 본인이 작업하고 싶은 파일이나 데이터의 “위치”를 직접 입력해야 하는 경우가 많은데, 그것을 일일이 적으려면 공간낭비 및 시간낭비가 상당합니다(제 얘깁니다…). R은 RSTUDIO가 “Import Dataset”이라는 콘솔을 이용하여 데이터를 R 로 불러올 수 있기는 합니다만, 그걸로는 부족할 때가 있습니다. 얘를 들어, 사용해야 하는 데이터 셋의 갯수가 여러개 일때, 일일이 콘솔로 불러 오는것도 귀찮아 집니다.

이와 같은 “귀찮은” 프로세스를 줄일 수 있는것이 setwd()' 입니다. 얘를 들어 제가 사용하고 싶은 데이터가C:\Users\imhyu\Desktop\Data Science\Study Group\data\`에 있을때, 아래와 같이

setwd("C:\\Users\\imhyu\\Desktop\\Data Science\\Study Group\\data\\")
getwd()
## [1] "C:/Users/imhyu/Desktop/Data Science/Study Group/data"

getwd()를 통해 현재의 working directory의 위치를 확인할 수 있습니다.

어떤 데이터를 가져올 수 있는가

단적으로 얘기하자면, 본인 역량대로… 아는 패키지의 갯수와 그 패키지들을 자유로히 사용할 수 있다면, 다른 오픈소스들 처럼 R 또한 거의 모든 종류의 데이터를 끌고 오는것이 가능합니다. 다음에는 STATA의 dta, SAS 등의 데이터를 가져오는 작업도 해보겠습니다.

만약 stata의 dta등의 파일들을 끌고 오는것에 관심이 있으시다면, foreign 패키지를 알아보시는 것도 좋을 겁니다. stata의 dta는 상당히 가벼운 데이터 종류이기에, 괜찮은 데이터파일이라고 생각합니다.

빅데이터에서 개인적으로 가장 가벼운 데이터의 종류는 메모장 파일입니다(txt). 보안에 문제가 많기는 합니다만, 모든 데이터 종류 중 메모장만큼 가벼운 데이터 종류는 보기 힘듭니다.

데이터 포멧

  • 데이터 포멧?
  • KDI에서는 대부분 Data Frame이라 부르는 데이터 포멧을 사용합니다. 하지만, 실제로는 다양한 데이터의 종류가 존재하는데, 주식시장에서는 xts라 부르는 시계열 데이터 포멧을 사용합니다. Econometrics나 왠만한 Survey 데이터는 거의다 Data Frame 또는 xts의 데이터 포멧을 사용하기에, 저희 스터디그룹은 Data Frame을 중심으로 다룰 겁니다. 시계열은 현재 KDI가 제공하는 QM이나 IRM 또는 Impact Evaluation 이 제공하는 정보로는 진행이 불가능하기에, 이 스터디에서는 간단한 모델만 가능하면 나중에 다루도록 하겠습니다.

Data Visualization (ggplot2)

  • ggplot으로 plot하기

ggplot2는 그래프를 만들때 가장 많이 사용하는 패키지입니다. Data Visualizing을 주로 하는 친구의 말에 의하면 ggplot은 마치 포토샵과 같은 레이어 구조를 띄고 있다고 합니다. ggplot2에서 그래프를 만들때 크게는 3개의 레이어로 나눕니다: 배경 설정(축) + 그래프(점, 막대, 선 등) + 설정(축 범위, 색, 표식 등)

# install.packages("ggplot2")

library(ggplot2)
  1. 배경 설정 - 말 그대로 배경만 설정합니다. 이 예에서는 mpg라는 R의 기본 데이터를 사용하겠습니다.
ggplot(data=mpg,aes(x=displ,y=hwy))

  1. +그래프
ggplot(data=mpg, aes(x=displ,y=hwy))+
  geom_point()

geom_point() 는 scatterplot, 즉 산점도를 그리는 커맨드입니다.

  • +설정추가

이번에는 x축의 범위를 임의로 설정하겠습니다.

ggplot(data=mpg,aes(x=displ,y=hwy))+
  geom_point()+
  xlim(3,6)

Tibble이란

Tibble은 현재 R 내에서 빠른 속도로 그 쓰임새가 늘고 있는 “데이터 타입” 입니다. 보통 STATA,SPSS,EXCEL, R 등의 통계언어들은 data frame(DF) 이라는 데이터 타입을 사용합니다(굉장히 efficient한 포멧입니다). 그런데 이러한 DF들의 단점은 DF를 call 할 경우, 데이터 전체를 프린트를 할 문제점이 있다는 겁니다 (잘못하면 컴퓨터나 프로그램이 삐져서 파업합니다).

그리고 DF는 각 variable이 무슨 타입인지를 데이터만 봐서는 모른다는 단점이 있습니다. STATA를 좀 쓰신 분들은 아시겠지만, data type에 따라서 사람 속터지게 만드는 경우가 가끔 있지요…(excel에서 currency 면 stata는 그걸 숫자가 아닌 단어로 인식하는 놀라운 인식능력을 갖추고 있습니다…왜???)

R 또한 그 정도는 아니지만 그래도 가끔씩 귀찮을 때가 좀 있습니다. 그런데 tibble은 이러한 점들을 여러모로 커버를 해줍니다.
