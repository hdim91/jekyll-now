What is R?

R is a high-level open source programming language and software environment for statistical computing and graphics that is supported by the R Foundation for Statistical Computing. The R language is widely used among statisticians and data miners for developing statistical software and data analysis. Polls, surveys of data miners, and studies of scholarly literature databases show that R's popularity has increased substantially in recent years. - Wikipedia

Source: Indeed

What is the advantage of using R?

Each programming language has its own pros and cons. R is not that much different from them. The advantage of R is that it is an open source programming language that is mainly developed for statistical analysis. In other words, R is free. However, it also means that you need to learn how R works and basic commands in order to use it fluently. In addition, because R's commands are mainly based on the vectorization, it is much easier to assign and call variables and to create customized functions for the statistical analysis. I will talk about this vectorization issue in the later slide.

What is the best timing for using R?

Honestly, there is no best time for using R specifically. However, if I have to state when is the best time, it would be when you need to conduct diverse statistical analysis. Lots of scholars and professionals in the field of statistics and computer programmings provide useful packages to conduct diverse analysis.

Vectorization: Why is it a big deal?

Since I am not a computer sicentist or computer engineer, I cannot explain thing technically or theoratically, but the advantage of vectorization is that when you use and assign data, you can manage the memory of your computer better. In other words, you can write a simple code to construct functions or dataset in R. For further information, http://www.noamross.net/blog/2014/4/16/vectorization-in-r–why.html explains the strength of vectorization well.

This is the beauty of R. In Python, if you want to vectorize your data and transform it into matrix, you need a package called "numpy," while R already has it in default.

Various Application

Comparing to SPSS, STATA, or SAS, R is indeed hard to learn since you need learn more commands comparing to these programs. However, the strength of open source programming languages is that they relatively have only few constraints. For example, the slide I created is made by the combination of LaTEX(I used Miktex) and RStudio.

Some R developers of R packages provide their e-book in online free. (e.g. ggplot2, caret, plm, etc) Therefore if you find an interesting package and if you have confidence that you can use that package for your own analysis, you can take a look at the information from online.
